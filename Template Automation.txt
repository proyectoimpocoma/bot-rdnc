2. Objetivo del negocio
Problema actual: Deben consular las rutas en la pagina de sictac y tener que responder una cadena de preguntas para obtener la información.
Objetivo de la automatización: Reducir la fricción para consultar en la pagina, agilizar como se obtienen los datos.
Resultado esperado: Un chatbot que lea en lenguaje natural, origen y destino de un viaje de carga, y devuelva los datos.

3. Descripción del proceso actual (AS-IS):
Pasos:
1. Ingresar a https://plc.mintransporte.gov.co/Runtime/empresa/ctl/SiceTAC/mid/417
2. Seleccionar la configuración del vehículo.
3. Seleccionar condiciones del vehículo -> [Cargado vacío]
4. Tipo de carga -> [General - Granel solido]
5. Origen: Ubicación? - Ciudad - Departamento.
6. Destino: Ubicación? - Ciudad - Departamento.
7. Via? Opcional valor por defecto
8. Horas cargue y descargue
9. Tomar el Costo Total del Viaje
10. Validar valor real en RNDC el precio del flete.
11. Con los mismo datos de vehiculo.
12. Mostrar precios comparados de las paginas.
13. Codigo de vehiculo.
14. Cada trayecto de vehiculo. precio de viaje unitario promediado.
15. Pendiente validar en tms, y comparar ruta.

13. Tecnologías sugeridas
Procesamiento de datos: Polar por la cantidad de datos.
Procesamiento de lenguaje natural: spaCy para reconocimiento de entidades nombradas (NER) en ciudades y departamentos colombianos; OpenAI GPT API para parsing avanzado de consultas en texto natural; NLTK o Transformers (Hugging Face) como alternativa gratuita para tokenización y clasificación.
Automatización web: Playwright o Selenium para scraping y automatización de navegadores web en Sictac y RNDC (manejo de formularios interactivos, selección de opciones y extracción de costos); requests y beautifulsoup para parsing HTML complementario.
Base de datos: Excel descargado de de RNDC. y datos de SiceTAC.
Correos / notificaciones: Conexion a chat Whatapp.
Plataforma de chatbot: WhatsApp (usando Twilio API), Telegram Bot API, y una página web con chat integrado (usando frameworks como Streamlit).
